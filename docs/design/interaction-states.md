# interaction-states.md

## 目的

本文档定义状态表达、交互反馈、错误提示和加载状态。

目标是让用户知道当前发生了什么，同时避免焦虑式表达。

---

## 任务状态

| 状态 | 视觉 |
|---|---|
| todo | 白色卡片 + neutral 标签 |
| done | success-soft 背景 + success 标签 |
| partial | warning-soft 背景 + warning 标签 |
| skipped | surface-soft 背景 + muted 标签 |

规则：

- todo 是正常状态，不应显得落后。
- done 可以有轻微正反馈。
- partial 表示仍有推进，不是失败。
- skipped 不使用红色，不做惩罚性表达。

---

## 能量状态

| 状态 | 视觉 |
|---|---|
| low | warm-soft 背景 + 温和提示 |
| normal | accent-soft 背景 + 常规提示 |
| high | success-soft 背景 + 推进提示 |

规则：

- low 不等于失败。
- low 状态应优先展示最低完成线。
- high 状态可以增加推进感，但不强行加任务。

---

## 计划状态

| 状态 | 视觉 |
|---|---|
| 可执行 | accent 标记 |
| 需要调整 | warning-soft 卡片 |
| 不可行 | danger-soft 小提示，不使用大面积红色 |

不可行计划提示必须给出可选动作：

- 延长 DDL
- 增加每日可用时间
- 缩小目标范围

---

## 加载状态

用于：

- 生成计划
- 保存复盘
- 重排任务
- 读取存储数据

规则：

- 使用轻量 skeleton 或简短文案。
- 不使用夸张动画。
- 不使用“正在预测你的命运”等玄学表达。
- 超过合理等待时间时，给出重试或 fallback 方案。

推荐文案：

```text
正在把目标拆成更容易开始的小步
```

---

## 成功反馈

成功反馈要短、明确、温和。

示例：

```text
已生成今天的任务
复盘已保存，明天的安排已调整
今天至少已经往前推了一步
```

规则：

- 不夸大结果。
- 不使用虚假安慰。
- 不制造连续打卡压力。

---

## 错误反馈

错误提示必须具体。

示例：

```text
截止日期不能早于今天
每日可用时间需要大于 0 分钟
计划暂时生成失败，已为你使用基础规则生成安排
```

规则：

- 错误文案不责备用户。
- 不展示技术细节、密钥、内部服务信息。
- 如果 AI 失败，提示 fallback，而不是直接中断。

---

## 空状态

空状态必须包含下一步行动。

示例：

```text
还没有目标
先写下一个想推进的目标，我来帮你拆成每天能做的小步。
```

规则：

- 不使用“你还没有完成任何事”。
- 不使用失败感表达。
- 不放大空白带来的压力。

---

## 可点击区域

- 所有主要按钮最小高度 44px。
- 可点击区域最小 44px。
- 状态标签如果可点击，必须有明显反馈。
- 不把小图标作为唯一点击入口。

---

## 表单状态

表单字段必须支持：

- default
- focus
- filled
- error
- disabled

错误提示应显示在字段下方。

禁用状态必须说明原因或通过上下文可理解。

---

## 动效规则

MVP 阶段动效保持克制。

允许：

- 轻微 fade
- 轻微 slide
- 按钮点击反馈
- 保存成功轻提示

禁止：

- 大面积粒子动画
- 复杂加载动画
- 频繁弹窗
- 强刺激打卡动效
