# 创建目标页设计规范

## 1. 页面定位

创建目标页负责创建 Goal。

负责：

- 收集目标名称。
- 收集截止日期。
- 收集每日可用时间。
- 收集补充说明。
- 收集个性化安排偏好。
- 点击生成计划后创建 Goal 并触发 Plan 生成。

不负责：

- 展示任务日历。
- 执行今日任务。
- 展示完整 Plan。
- 处理当天复盘。

## 2. 当前问题

- 页面主体不应重复「创建目标」标题。
- 原始表单容易像复杂设置页。
- 个性化偏好需要展开，但要保持它是偏好而非硬性心理测评。
- 塔罗、每日关键词只能影响表达风格，不能影响任务安排科学性。

## 3. 页面目标

用户打开创建目标页后，首先应理解：

```text
我只需要设定一个目标，系统会据此生成计划。
```

## 4. 目标页面结构

```text
顶部导航栏：创建目标

页面主体：
  轻量引导区
  目标名称步骤卡
  截止日期步骤卡
  每日可用时间步骤卡
  补充说明卡
  个性化安排偏好卡
  生成计划按钮
  底部安全提示

底部 TabBar：
  今日 / 日历 / 创建 / 我的
```

## 5. 核心组件说明

| 组件 | 展示内容 | 交互 | 数据来源 | 空状态 | 异常状态 | 是否 MVP 必需 |
|---|---|---|---|---|---|---|
| 轻量引导区 | 简短说明和轻图标 | 无 | 静态文案 | 可隐藏 | 不得变成长说明 | MVP 必须 |
| 目标名称卡 | 输入框、字数提示、辅助说明 | 输入目标名 | 用户输入 | 空时按钮禁用 | 无效字符或过长提示 | MVP 必须 |
| 截止日期卡 | 日期选择 | 打开日期选择器 | 用户输入 | 空时按钮禁用 | 早于今天时提示 | MVP 必须 |
| 每日可用时间卡 | 15/30/45/60/自定义 | 选择或输入分钟数 | 用户输入 | 可默认 30 分钟 | 非法分钟数提示 | MVP 必须 |
| 补充说明卡 | 可选多行输入 | 输入说明 | 用户输入 | 可为空 | 过长提示 | MVP 必须 |
| 个性化安排偏好卡 | 工作时段、能量、计划强度、仪式感 | 选择偏好 | UserProfile / 创建输入 | 使用默认值 | 非法状态回落默认值 | MVP 必须 |
| 生成计划按钮 | 主行动 | 创建 Goal、触发 Plan 生成 | 表单状态 | 禁用 | 生成失败不清空表单 | MVP 必须 |

## 6. 状态设计

- 初始状态：目标名称、截止日期为空；每日可用时间默认 30 分钟或未选；生成计划按钮禁用。
- 填写中：字段实时更新，错误显示在字段附近。
- 可提交：目标名称、截止日期、每日可用时间均有效。
- 生成中：按钮 loading，防止重复点击。
- 生成失败：显示轻量错误，保留表单内容。
- 生成成功：创建 Goal，并触发 Plan 生成。

## 7. 数据依赖

完整 Goal、Plan、Stage、Task 边界默认引用 `docs/architecture/goal-plan-task-state-model.md`。

| 数据 | 字段 | 用途 | 状态 |
|---|---|---|---|
| Goal | title | 目标名称 | MVP 必须 |
| Goal | deadline | 截止日期 | MVP 必须 |
| Goal | dailyAvailableMinutes | 每日可用时间 | MVP 必须 |
| Goal | description | 补充说明 | MVP 必须 |
| UserProfile | workStyle | 偏好工作时段 | MVP 必须 |
| UserProfile | energyLevel | 当前能量状态 | MVP 必须 |
| UserProfile | ritualPreference | 仪式感偏好 | MVP 必须 |
| Goal / PlanInput | planIntensity | 计划强度 | 建议新增 |
| Goal / PlanInput | ritualElement | 每日关键词 / 塔罗灵感 / 都不需要 | 建议新增 |

待确认：计划强度应归入 Goal、UserProfile 还是 PlanInput。

## 8. 交互跳转

- 生成计划：创建 Goal，并触发 Plan 生成。
- 生成成功后推荐跳转任务日历页。
- 如果当前产品更强调立即执行，可跳转今日任务页；待确认。
- 个性化偏好默认展开。
- 仪式感元素只影响表达风格，不影响任务安排科学性。

## 9. 视觉要求

- 使用步骤式卡片。
- 目标名称、截止日期、每日可用时间是前三个核心输入。
- 每日可用时间使用矩形胶囊按钮。
- 个性化偏好默认展开。
- 生成计划按钮使用主色。
- 底部提示保持轻量。
- 不使用长页面说明。

## 10. 不要做的事情

- 不在页面主体重复创建目标大标题。
- 不直接展示任务日历。
- 不直接执行任务。
- 不做复杂设置页。
- 不强制填写 MBTI。
- 不让塔罗、MBTI、每日关键词决定任务安排。
- 不要求用户填写完整总工作量。

## 11. 验收标准

- 页面使用步骤式卡片。
- 包含目标名称、截止日期、每日可用时间、补充说明、个性化偏好。
- 个性化偏好默认展开。
- 个性化偏好包含偏好工作时段、当前能量状态、计划强度、仪式感元素。
- 仪式感元素明确只影响表达风格。
- 生成计划按钮在必填项未完成时禁用。
- 点击生成计划后创建 Goal 并触发 Plan 生成。
- 创建目标页不展示任务日历，不执行任务。
