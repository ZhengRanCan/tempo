# 任务日历页设计规范

## 1. 页面定位

任务日历页是目标计划板。

负责：

- 展示当前目标计划卡。
- 展示当前目标整体进度。
- 展示时间压力和延期风险。
- 展示未来 7 天计划概览。
- 展示长期目标的阶段安排。
- 点击某一天后展示当天任务列表。
- 提供调整计划入口。

不负责：

- 承担今日任务的主要执行。
- 展示所有任务的无结构长列表。
- 进行复杂塔罗或 MBTI 分析。

## 2. 当前问题

- 页面容易变成普通任务列表。
- 目标名和统计数字如果裸露展示，会缺少上下文。
- 长期目标不应一次性细化未来所有天的任务。
- 进度必须是当前目标整体进度，不是单个任务进度。

## 3. 页面目标

用户打开任务日历页后，首先应理解：

```text
当前目标是什么、整体进度如何、未来 7 天如何安排、是否有时间风险。
```

## 4. 目标页面结构

```text
顶部导航栏：任务日历

页面主体：
  当前目标计划卡
  计划进度 / 时间压力卡
  未来 7 天计划概览
  选中日期任务列表
  AI 计划建议卡
  调整计划入口

底部 TabBar：
  今日 / 日历 / 创建 / 我的
```

## 5. 核心组件说明

| 组件 | 展示内容 | 交互 | 数据来源 | 空状态 | 异常状态 | 是否 MVP 必需 |
|---|---|---|---|---|---|---|
| 当前目标计划卡 | 当前目标、截止日期、剩余天数、任务数、剩余时间、计划状态 | 切换目标或管理目标 | Goal、Plan | 无目标时显示创建目标引导 | Goal 缺失 deadline 时提示数据异常 | MVP 必须 |
| 进度 / 时间压力卡 | 进度条、完成任务数、完成天数、剩余时间、剩余天数 | 查看 AI 建议 | Plan、Task 汇总 | 无计划时提示生成计划 | 任务时长缺失时使用任务数量算法 | MVP 必须 |
| 日期排布视图 | 未来 7 天、任务数、预计用时、状态 | 选择日期、周/月切换 | DailyPlan、Stage、Task | 无任务日显示缓冲日 | 任务过多时标记计划偏紧 | MVP 必须 |
| 选中日期任务列表 | 当天任务、最低完成线、优先级、状态、操作 | 查看详情、调整日期、移到今天、标记完成 | Task | 当天无任务显示缓冲说明 | 延期任务显示风险标签 | MVP 必须 |
| AI 计划建议卡 | 简短计划建议 | 查看详情预留 | TodaySuggestion / AiTodaySuggestion | 无建议时隐藏 | AI 失败时显示 fallback | MVP 必须 |
| 调整计划入口 | 修改时间、DDL、强度、重新生成计划 | 打开弹窗或页面 | Goal、Plan | 无目标时隐藏 | 重排失败时提示不清空计划 | MVP 必须 |

## 6. 状态设计

- 无目标：显示“还没有目标”，按钮「创建目标」。
- 有目标但无计划：显示当前目标，按钮「生成计划」。
- DDL 7 天以内：展示未来 7 天计划，强调剩余天数。
- DDL 超过 7 天：展示未来 7 天详细任务，并提供周视图 / 月视图或长期阶段入口。
- 某一天无任务：显示“这天暂无任务 / 缓冲日”。
- 某一天任务过多：标记“任务偏重”或“计划偏紧”。
- 有延期任务：日期视图标记风险，任务列表显示延期状态，提供重排入口。
- 目标已完成：显示完成状态，保留历史任务，弱化调整计划入口。

## 7. 数据依赖

日历页主要读取 Goal、Plan、Stage、Task。

完整 Goal、Plan、Stage、Task 边界默认引用 `docs/architecture/goal-plan-task-state-model.md`。

| 数据 | 字段 | 用途 | 状态 |
|---|---|---|---|
| Goal | id | 当前目标定位 | MVP 必须 |
| Goal | title | 当前目标名称 | MVP 必须 |
| Goal | deadline | 计算剩余天数 | MVP 必须 |
| Goal | status | 目标状态 | 建议新增 |
| Plan | goalId | 关联当前目标 | MVP 必须 |
| Plan | progressPercent | 整体进度 | 建议新增 |
| Plan | planStatus | 计划状态 | 建议新增 |
| Plan | remainingEstimatedMinutes | 剩余时间 | 建议新增 |
| Stage | title/startDate/endDate/status | 远期阶段安排 | 后续扩展 |
| Task | date 或 scheduledDate | 日期排布 | MVP 必须 |
| Task | estimatedMinutes | 日任务总时长 | MVP 必须 |
| Task | status | 日期状态和任务状态 | MVP 必须 |
| Task | minimumLine | 任务列表展示 | MVP 必须 |

待确认：当前实现中 `DailyPlan` 是否承担 Plan 的一部分职责，后续是否需要独立 `Plan` 和 `Stage` 模型。

## 8. 交互跳转

- 切换目标：进入目标选择器；如果暂不支持多目标，隐藏或弱化入口。
- 周视图 / 月视图：同页切换；月视图可先 UI mock。
- 点击日期：刷新下方选中日期任务列表。
- 查看全部任务：展示该日期全部任务。
- 调整日期：打开任务日期调整弹窗，待确认。
- 移到今天：更新任务日期或预留入口，待确认。
- 调整计划：打开计划调整弹窗或二级页。

## 9. 视觉要求

- 第一张卡是当前目标计划卡。
- 首屏应看到目标计划卡、进度卡或日期概览开头。
- 日期小卡必须区分有任务、无任务、完成、缓冲、风险。
- AI 计划建议卡内容简短。
- 不使用信息密度过高的传统月历。
- 不裸露目标名或裸数字。

## 10. 不要做的事情

- 不把所有任务直接堆在首页。
- 不把日历页做成今日执行页。
- 不用大段 AI 解读。
- 不做复杂塔罗牌阵。
- 不把塔罗、MBTI、每日关键词作为计划依据。
- 不把任务完成进度误写成目标完成。

## 11. 验收标准

- 顶部没有重复标题区。
- 页面第一张卡是当前目标计划卡。
- 当前目标名称、截止日期、剩余天数、总任务数、预计剩余时间都有上下文。
- 计划进度显示当前目标整体进度。
- 默认展示未来 7 天计划概览。
- DDL 超过 7 天时有周视图 / 月视图或长期阶段入口。
- 点击某一天后，下方展示该日期任务列表。
- 日期视图能区分有任务、无任务、已完成、缓冲日、延期风险。
- 有简短 AI 计划建议。
- 有调整计划入口。
