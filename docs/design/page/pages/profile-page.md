# 我的页设计规范

## 1. 页面定位

我的页是我的目标与偏好页。

负责：

- 展示当前目标概览。
- 管理目标入口。
- 展示默认安排偏好。
- 展示 AI 表达与仪式感偏好。
- 展示最近推进。
- 提供设置入口。

不负责：

- 当天任务执行。
- 展示完整任务日历。
- 开始专注。
- 把今日任务、任务日历入口重复做成按钮。

## 2. 当前问题

- 页面主体重复「我的」胶囊标签和个人状态标题。
- 当前目标卡片信息不足。
- 有目标时过度突出创建新目标。
- 默认安排偏好和 AI 表达偏好混在一起。
- 今日任务、任务日历入口与底部 TabBar 重复。

## 3. 页面目标

用户打开我的页后，首先应理解：

```text
我当前在推进哪个目标，以及我的默认安排和表达偏好是什么。
```

## 4. 目标页面结构

```text
顶部导航栏：我的

页面主体：
  用户问候区
  当前目标卡片 / 无目标引导卡
  默认安排偏好卡
  AI 表达与仪式感偏好卡
  最近推进卡
  设置入口列表

底部 TabBar：
  今日 / 日历 / 创建 / 我的
```

参考图：`../reference_image/我的.png`。

## 5. 核心组件说明

| 组件 | 展示内容 | 交互 | 数据来源 | 空状态 | 异常状态 | 是否 MVP 必需 |
|---|---|---|---|---|---|---|
| 用户问候区 | 默认头像、短问候 | 进入个人资料或设置预留 | UserProfile / 静态文案 | 使用默认头像和通用文案 | 用户资料缺失时不阻塞 | MVP 必须 |
| 当前目标卡 | 目标名、DDL、剩余天数、进度、计划状态 | 查看计划、管理目标 | Goal、Plan、Task 汇总 | 无目标时显示创建目标引导 | 进度缺失时用任务数量兜底 | MVP 必须 |
| 默认安排偏好卡 | 每日可用时间、偏好时段、计划强度、能量状态 | 编辑偏好 | UserProfile | 使用默认值 | 保存失败提示 | MVP 必须 |
| AI 表达与仪式感偏好卡 | 建议风格、每日关键词、塔罗灵感、MBTI | 编辑表达偏好 | UserProfile、AiPreference | 未填写显示未设置 | 不得暗示预测能力 | MVP 必须 |
| 最近推进卡 | 本周完成、累计专注、连续推进 | 查看记录预留 | Stats / Review | 无数据时隐藏或显示 0 | 数据不足时标记 UI mock | UI mock / 后续扩展 |
| 设置入口列表 | 目标管理、复盘记录、偏好设置、关于反馈 | 跳转页面或 toast | 静态配置 | 始终显示 | 未实现入口说明占位 | MVP 必须 |

## 6. 状态设计

- 有当前目标：显示当前目标卡、默认安排偏好、AI 表达偏好、最近推进、设置入口。
- 无当前目标：当前目标卡替换为无目标引导，突出「创建目标」。
- 目标有延期风险：目标卡状态显示“有延期风险”，使用橙色提示，操作为「查看计划」「调整计划」。
- 目标已完成：显示已完成状态，可提供「查看记录」「创建新目标」。
- 偏好缺失：使用默认值，不阻塞页面。
- 最近推进无真实数据：可隐藏或标记 UI mock。

## 7. 数据依赖

完整 Goal、Plan、Stage、Task 边界默认引用 `docs/architecture/goal-plan-task-state-model.md`。

| 数据 | 字段 | 用途 | 状态 |
|---|---|---|---|
| Goal | title | 当前目标名称 | MVP 必须 |
| Goal | deadline | 截止日期 | MVP 必须 |
| Goal | status | 目标状态 | 建议新增 |
| Goal | progressPercent | 当前目标进度 | 建议新增 |
| Plan | status | 计划状态 | 建议新增 |
| Task | completed / total | 进度兜底计算 | MVP 必须 |
| UserProfile | preferredFocusMinutes | 默认专注时长 | MVP 必须 |
| UserProfile | workStyle | 偏好时段 | MVP 必须 |
| UserProfile | energyLevel | 默认能量状态 | MVP 必须 |
| UserProfile | ritualPreference | 仪式感偏好 | MVP 必须 |
| AiPreference | suggestionStyle | AI 建议风格 | 建议新增 |
| Stats | weeklyCompletedTaskCount | 最近推进 | UI mock / 后续扩展 |

## 8. 交互跳转

- 查看计划：跳转任务日历页。
- 管理目标：进入目标管理页或目标详情弹窗，待确认。
- 创建目标：跳转创建目标页。
- 编辑默认安排偏好：打开偏好设置页或弹窗，待确认。
- 编辑 AI 表达偏好：打开表达偏好页或弹窗，待确认。
- 设置入口：未实现时可使用占位提示，但必须回传说明。

## 9. 视觉要求

- 当前目标卡片是核心卡片。
- 有目标时主操作是「查看计划」「管理目标」。
- 无目标时突出「创建目标」。
- 默认安排偏好和 AI 表达偏好分开。
- AI 表达卡必须包含边界说明。
- 设置入口列表简洁，不做大卡堆叠。

## 10. 不要做的事情

- 不在页面主体重复我的页标题。
- 不放「今日任务」「任务日历」重复入口。
- 不在我的页处理当天任务执行。
- 不把 MBTI、塔罗、每日关键词放入默认安排硬参数。
- 不强迫用户填写头像、昵称、MBTI。
- 不为了 UI 展示新增大量无用字段。

## 11. 验收标准

- 页面主体不出现「我的」胶囊标签、个人状态大标题和说明副标题。
- 删除「今日任务」「任务日历」重复跳转按钮。
- 页面第一张核心卡片是当前目标或无目标引导。
- 有目标时主操作是「查看计划」「管理目标」。
- 无目标时突出「创建目标」。
- 默认安排偏好包含每日可用时间、偏好时段、计划强度、能量状态。
- AI 表达与仪式感偏好单独成卡。
- MBTI、塔罗、每日关键词不作为预测依据。
- 最近推进最多展示 2-3 个核心指标。
