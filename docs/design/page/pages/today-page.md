# 今日任务页设计规范

## 1. 页面定位

今日任务页是执行台。

负责：

- 告诉用户今天最应该做什么。
- 突出今日重点任务。
- 展示今日任务数量和查看全部入口。
- 允许用户调整今日状态。
- 提供轻量 AI 今日建议。
- 提供晚间复盘入口。

不负责：

- 展示完整目标计划。
- 展示长期阶段安排。
- 管理目标。
- 生成复杂塔罗牌阵或长篇 AI 解读。

## 2. 当前问题

- 原始需求明确要求今日页改成执行页，而不是信息展示页。
- 今日页容易和任务日历页混淆，必须明确只服务当天执行。
- 多任务时不能把所有任务都堆在首屏。
- 今日状态选择需要保持轻量，复杂配置应放到管理入口。
- AI 今日建议容易变成长文，需要限制为短内容。

## 3. 页面目标

用户打开今日任务页后，首先应理解：

```text
今天最重要的一件事是什么，最低做到什么程度就算推进。
```

## 4. 目标页面结构

```text
顶部导航栏：今日任务

页面主体：
  今日重点任务卡
  今日状态卡
  AI 今日建议卡
  晚间复盘入口

底部 TabBar：
  今日 / 日历 / 创建 / 我的
```

页面主体不得重复「今日任务」大标题、胶囊标签或长副标题。

## 5. 核心组件说明

| 组件 | 展示内容 | 交互 | 数据来源 | 空状态 | 异常状态 | 是否 MVP 必需 |
|---|---|---|---|---|---|---|
| 今日重点任务卡 | 今日重点任务、目标上下文、预计时长、优先级、最低完成线 | 开始专注、标记完成 | Task、Goal、TodaySuggestion | 今日无任务时显示 EmptyState | Task 缺失最低完成线时显示兜底文案 | MVP 必须 |
| 今日任务数量入口 | 今日共 N 个任务、查看全部、其余任务摘要 | 进入今日全部任务页或底部抽屉 | 今日 Task 列表 | N=0 时隐藏 | 待确认：全部任务页或抽屉采用哪种方案 | MVP 必须 |
| 今日状态卡 | 当前能量、可用时间、当前状态、管理入口 | 修改状态、进入管理 | UserProfile、今日临时状态 | 使用默认状态 | 保存失败时提示不清空当前选择 | MVP 必须 |
| AI 今日建议卡 | 1-3 条短建议、今日关键词 | 查看详情预留 | TodaySuggestion、AiTodaySuggestion、TarotDraw 可选 | 无建议时隐藏或显示基础建议 | AI 失败时显示 fallback 建议 | MVP 必须 |
| 晚间复盘入口 | 复盘说明、去复盘按钮 | 跳转 review 页面 | 今日 Task 列表、DailyReview | 今日无任务时弱化显示 | 已复盘时显示已完成状态 | MVP 必须 |

### 今日重点任务卡

MVP 必须展示：

- 小标签：今日重点任务。
- 当前目标名称。
- 说明文案：当前最值得推进的一步。
- 今日主任务标题。
- 元信息：预计时长、优先级。
- 最低完成线。
- 主按钮：开始专注。
- 次操作：标记完成。

多任务时，卡片内部必须增加：

```text
今日共 N 个任务 · 查看全部
```

可附带一行短预览：

```text
其余任务：整理研究背景、补充参考文献、确认研究问题
```

### 今日状态卡

MVP 必须包含三行快速选项：

- 当前能量：低 / 普通 / 高 / 更多。
- 可用时间：15 分钟 / 30 分钟 / 60 分钟 / 自定义。
- 当前状态：有点焦虑 / 正常 / 想冲刺。

默认值：

- 当前能量：普通。
- 可用时间：30 分钟。
- 当前状态：正常。

右上角必须有「管理」入口。

管理入口内容建议：

- 常用可用时间。
- 默认可用时间。
- 常用能量状态。
- 常用工作时段。

### AI 今日建议卡

MVP 必须：

- 标题为 AI 今日建议。
- 可显示今日关键词。
- 正文最多 2-3 条。
- 底部说明塔罗灵感仅用于提醒节奏，不作为预测依据。

示例：

```text
- 先完成重点任务的最低完成线。
- 其余任务可在状态允许时继续推进。
- 如果卡住，就记录问题，晚上复盘时重排。
```

### 晚间复盘入口

MVP 必须轻量展示：

- 标题：晚间复盘。
- 说明：晚上记录完成情况，我会自动帮你重排明天。
- 按钮：去复盘。

## 6. 状态设计

- 无目标：显示创建目标空状态，主按钮为「创建目标」。
- 有目标但今日无任务：显示“今天暂无任务”，可提供「查看日历」。
- 今日 1 个任务：直接作为今日重点任务展示。
- 今日多个任务：首屏只突出 1 个主任务，并显示「今日共 N 个任务 · 查看全部」。
- 任务已完成：状态显示为已完成；主按钮不再显示「开始专注」，或显示已完成态。
- 当前能量 low：优先展示最低完成线，文案降低压力。
- 当前可用时间不足：AI 今日建议提醒先做最低完成线。
- 晚间已复盘：复盘入口显示已完成或查看复盘记录。
- AI 不可用：使用 deterministic fallback，不中断页面。

## 7. 数据依赖

主要读取 Task 数据。

完整 Goal、Plan、Stage、Task 边界默认引用 `docs/architecture/goal-plan-task-state-model.md`。

| 数据 | 字段 | 用途 | 状态 |
|---|---|---|---|
| Task | id | 操作任务 | MVP 必须 |
| Task | goalId | 关联目标 | MVP 必须 |
| Task | title | 今日任务标题 | MVP 必须 |
| Task | estimatedMinutes | 展示预计时长 | MVP 必须 |
| Task | priority | 选择今日重点任务 | MVP 必须 |
| Task | status | 展示完成状态 | MVP 必须 |
| Task | minimumLine | 展示最低完成线 | MVP 必须 |
| Task | type | 重点 / 普通 / 备用 | 建议新增 |
| Goal | title | 当前任务所属目标上下文 | MVP 必须 |
| UserProfile | energyLevel | 今日状态默认值 | MVP 必须 |
| TodayState | availableMinutes | 今日可用时间 | 建议新增 |
| TodayState | mood/status | 当前状态，如焦虑、正常、想冲刺 | 建议新增 |
| TodaySuggestion | dailyKeyword | 今日关键词 | UI mock / 后续扩展 |
| TodaySuggestion | reorderedTaskIds | 今日任务排序 | 后续扩展 |

## 8. 交互跳转

- 开始专注：进入专注流程或保留为 UI mock，待确认。
- 标记完成：更新 Task 状态，或进入复盘前置确认，待确认。
- 查看全部：优先进入今日全部任务页；如路由暂不适合，可用底部抽屉，需回传说明。
- 今日全部任务页：建议路径 `pages/today-all/index`，待确认。
- 管理今日状态：打开状态管理页或弹窗，待确认。
- 去复盘：跳转 review 页面。

## 9. 视觉要求

- 第一张卡必须是今日重点任务卡。
- 今日重点任务卡可以比其他卡更突出。
- 最低完成线使用浅紫背景块。
- AI 建议不超过 3 条短句。
- 晚间复盘入口轻量展示，不抢首屏。
- 不使用红色表达普通未完成。
- 首屏应能看到今日重点任务、查看全部入口，以及状态调整卡的一部分或完整状态卡。

## 10. 不要做的事情

- 不展示完整目标计划。
- 不把未来 7 天任务堆到今日页。
- 不在页面主体重复今日任务标题。
- 不写大段 AI 解读。
- 不把塔罗、MBTI、每日关键词作为科学预测依据。
- 不在今日页做目标管理。

## 11. 验收标准

- 页面第一张卡是今日重点任务。
- 首页只突出一个主任务。
- 多任务时显示「今日共 N 个任务 · 查看全部」。
- 点击「查看全部」能看到今天所有任务，并且主任务和其他任务有区分。
- 今日状态卡包含当前能量、可用时间、当前状态和管理入口。
- 可用时间支持「自定义」。
- 当前能量支持「更多」或通过「管理」扩展。
- AI 今日建议为短内容。
- 晚间复盘入口轻量展示。
- 今日页不展示完整目标计划。
- 今日页主要读取 Task 数据。
- 无目标、无今日任务、多个任务、low 能量状态均有明确展示。
