# F20 页面壳与组件基线设计合约

## 目的

本文档是 `F20` 的执行级设计合约，用于把「页面壳与全局组件基线」从抽象描述拆成可执行检查项。

`F20` 只建立 UI 改造的公共基础，不重做四个 Tab 页的信息结构。今日任务、任务日历、创建目标、我的页的完整页面重组分别由 `F21`、`F22`、`F23` 完成。

## 阅读顺序

执行 `F20` 前必须按顺序阅读：

1. `docs/harness/DESIGN.md`
2. `docs/design/README.md`
3. `docs/design/visual-system.md`
4. `docs/design/components.md`
5. `docs/design/interaction-states.md`
6. `docs/design/content-rules.md`
7. `docs/design/page/00-ui-redesign-overview.md`
8. `docs/design/page/01-page-layout-contract.md`
9. 本文档

如涉及具体页面首屏内容，只能少量参考对应页面文档，不得在 `F20` 内提前完成 `F21-F23` 的页面级重组。

## F20 的定位

`F20` 是 UI 改造的基线任务，目标是让后续页面改造有稳定的外壳、间距、组件语义和可复用样式。

`F20` 应完成：

- 四个 Tab 主页面和复盘页使用一致的页面外壳。
- 页面主体移除重复的大标题区、胶囊标签区和长副标题区。
- 底部 TabBar 只承担主页面切换。
- 全局卡片、按钮、标签、空状态、任务卡、能量选择器的样式语义统一。
- 现有页面数据读取路径保持不变。
- 页面可以继续显示当前已有信息，但展示方式需要收敛到统一组件和低压力视觉规则。

`F20` 不应完成：

- 不重排今日任务页为完整执行台。
- 不重排任务日历页为完整计划板。
- 不重排创建目标页为完整步骤流。
- 不重排我的页为完整目标与偏好中心。
- 不修改 Goal、Plan、Stage、Task、DailyReview、UserProfile 的数据结构。
- 不修改 storage key、planner、replanner、AI、tarot 服务逻辑。

## 页面壳规则

### AppPageHeader

`AppPageHeader` 负责页面顶部识别，不由页面主体重复承担。

MVP 必须：

- 支持展示当前页面名称。
- 支持一行短提示文案，文案必须短，不超过一行优先。
- 支持页面级轻量操作位，只有确有必要时使用。
- 视觉上与页面主体保持紧凑衔接，不制造大块顶部空白。

不得：

- 在页面主体再写同名大标题，例如顶部已经显示「今日任务」，主体不再出现大号「今日任务」。
- 在 header 下方追加无业务价值的胶囊标签。
- 把 AI、塔罗、MBTI、每日关键词做成顶部主视觉。

### 页面主体

MVP 必须：

- 导航栏下方直接进入当前页面核心内容。
- 页面主体顶部间距保持紧凑，建议使用 `24rpx-32rpx`。
- 主体只保留必要的内容分组，不额外增加营销式介绍区。
- 每个页面首屏最多一个视觉重点卡片。

不得：

- 使用大块空白撑出标题区。
- 使用大面积渐变、强阴影、玻璃拟态。
- 在页面主体重复 TabBar 已有入口。
- 把多个卡片嵌套成卡片套卡片。

## TabBar 协同规则

四个 Tab 主入口为：

- 今日任务
- 任务日历
- 创建目标
- 我的

MVP 必须：

- TabBar 只作为主页面切换入口。
- 页面主体不得再放置同等级的「去今日任务」「去任务日历」「去创建目标」「去我的」大按钮。
- 非 Tab 页面，例如复盘页、今日全部任务页、目标管理页，才可以作为页面内操作入口。

## F20 中各页面允许改到什么程度

### 今日任务页

允许：

- 接入统一页面壳。
- 移除主体重复标题。
- 使用统一 `TodayFocusCard`、`TaskCard`、`EmptyState` 样式外壳。
- 保留当前已有今日任务、今日建议、复盘入口等数据读取逻辑。

不得：

- 在 `F20` 内完整重排为「一个主任务 + 今日共 N 个任务」结构。
- 新增今日全部任务页。
- 改写 today-suggestion 业务规则。

后续由 `F21` 完成：

- 今日执行台信息结构。
- 多任务时的查看全部入口。
- 今日状态卡和短 AI 建议。

### 任务日历页

允许：

- 接入统一页面壳。
- 移除主体重复标题。
- 使用统一 `TaskCard`、`EmptyState`、状态标签样式。
- 保留当前已有日历/计划数据读取逻辑。

不得：

- 在 `F20` 内完整重排为目标计划板。
- 新增完整周/月视图。
- 改写 PlanBundle、Stage、Task 的 selector 逻辑。

后续由 `F22` 完成：

- 当前目标计划卡。
- 整体进度。
- 时间压力。
- 近 7 天 Task 与远期 Stage 的完整展示。

### 创建目标页

允许：

- 接入统一页面壳。
- 移除主体重复标题。
- 表单卡片、按钮、选项样式改为全局组件规则。
- 保留当前创建 Goal、保存偏好、生成 PlanBundle 的业务路径。

不得：

- 在 `F20` 内完整重排为步骤式创建流。
- 修改表单字段含义。
- 修改 planner 调用和 storage 写入路径。

后续由 `F23` 完成：

- 步骤式卡片组织。
- 个性化偏好默认展开。
- 仪式感元素边界表达。

### 我的页

允许：

- 接入统一页面壳。
- 移除主体重复标题。
- 使用统一卡片、空状态、偏好组选项样式。
- 保留当前目标、偏好、进度信息的读取逻辑。

不得：

- 在 `F20` 内完整重排为目标与偏好中心。
- 新增完整设置页。
- 把今日任务和任务日历做成重复强入口。

后续由 `F23` 完成：

- 当前目标卡。
- 默认安排偏好。
- AI 表达与仪式感偏好。
- 最近推进和设置入口。

### 复盘页

允许：

- 接入统一页面壳。
- 使用统一任务卡、状态标签、按钮和表单样式。
- 保持当前 DailyReview 保存和重排路径。

不得：

- 修改 review 数据结构。
- 修改 replanner 策略。
- 新增复杂复盘分析页。

## 组件基线

### TaskCard

`TaskCard` 是跨页面复用的任务信息外壳。

MVP 必须：

- 支持任务标题。
- 支持预计用时。
- 支持最低完成线。
- 支持状态标签。
- 支持一个主操作或低权重操作位。
- 支持今日页和日历页以不同权重使用同一外壳。

不得：

- 在组件内部写死今日页或日历页专属文案。
- 同一张任务卡放多个强主按钮。
- 用红色表达普通未完成或跳过。

### TodayFocusCard

`TodayFocusCard` 在 `F20` 只建立可复用外壳，不完成今日页最终信息结构。

MVP 必须：

- 能承载一个重点任务。
- 视觉权重高于普通 `TaskCard`。
- 仍遵守低压力视觉规则。

不得：

- 在 `F20` 中强行决定多任务选择算法。
- 写入 AI 决策逻辑。

### EmptyState

MVP 必须：

- 包含温和标题。
- 包含一句具体说明。
- 包含一个明确下一步动作。

不得：

- 只显示「暂无数据」。
- 使用责备式文案。
- 使用大面积插画抢占首屏。

### EnergySelector

MVP 必须：

- 使用统一选中态。
- 能表达低/中/高或等价能量状态。
- 选项文案短，适合移动端。

不得：

- 将 MBTI、塔罗、每日关键词混入能量状态。
- 把选项写成长解释文本。

## 视觉执行规则

具体 token 以 `docs/design/visual-system.md` 和 `docs/design/components.md` 为准。

F20 执行时必须遵守：

- 页面背景使用低压力浅色画布。
- 卡片使用 `surface + border`，默认不使用强阴影。
- 主按钮使用统一 accent 样式。
- 状态颜色必须配合文字，不单靠颜色表达。
- 单页强强调色面积不超过约 20%。
- 普通未完成、跳过、低能量不得使用高压红色。
- 不使用大面积渐变、玻璃拟态、强投影、高饱和背景。

## 内容文案规则

MVP 必须：

- 页面内文案短句优先。
- 空状态必须告诉用户下一步能做什么。
- AI、塔罗、MBTI、每日关键词只能作为轻量表达风格或仪式感，不作为科学预测或任务决策依据。
- 错误和不可行状态要给出可操作下一步。

不得：

- 输出大段 AI 解读。
- 用「你失败了」「严重落后」等高压力文案。
- 把仪式感内容放到首屏主视觉中心。

## 数据与服务边界

F20 不改变数据真相来源。

MVP 必须：

- 页面继续通过现有 services、selector 或 adapter 读取展示数据。
- 页面层不得直接拼装 `Plan + Stage + Task` 底层存储结构。
- 不新增 storage key。
- 不删除 legacy adapter。
- 不修改 planner、replanner、today-suggestion、ai-client、tarot 的业务逻辑。

如实现时发现组件需要缺失字段：

- 先用现有字段降级展示。
- 在 `docs/progress.md` 记录缺口。
- 不在 `F20` 内扩展模型字段。

## 测试与验收

自动验证至少覆盖：

- `tests/ui-components.test.ts`：组件文件、关键样式类、基础结构存在。
- `tests/navigation-shell.test.ts`：四个 Tab 主入口仍存在，顺序不被破坏。
- `tests/today.test.ts`：今日页仍可读取当前数据路径。
- `tests/plan-calendar.test.ts`：日历页仍可读取当前数据路径。
- `tests/review.test.ts`：复盘页保存路径不被破坏。

人工冒烟至少检查：

- 打开今日、日历、创建、我的四个 Tab，页面主体没有重复大标题。
- 顶部导航和主体内容之间没有大块空白。
- TabBar 已有入口没有在页面主体重复做成大按钮。
- 无目标、无计划、无今日任务时，空状态有具体下一步动作。
- 有任务数据时，任务卡、按钮、标签视觉一致。
- 普通未完成、跳过、低能量没有用高压红色表达。

## 完成标准

`F20` 完成时应满足：

- 四个 Tab 主页面和复盘页都接入统一页面壳或等价页面壳规则。
- 全局组件样式基线可被 `F21-F23` 继续复用。
- 页面现有业务路径保持可用。
- 没有提前实现 `F21-F23` 的页面级重组。
- `npm.cmd run verify:harness`、`npm.cmd run verify:static`、`npm.cmd run verify:system` 和 F20 指定测试通过。
