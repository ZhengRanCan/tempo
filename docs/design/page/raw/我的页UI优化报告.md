# 我的页 UI 优化报告

## 1. 报告背景

当前小程序已有四个底部 Tab 页面：今日、日历、创建、我的。

前面已经确定过一个基本原则：顶部导航栏负责显示当前页面名称，底部 TabBar 负责页面切换。因此，「我的」页页面主体不需要再出现重复的胶囊标签、大标题和说明副标题。

当前「我的」页的问题不是单个组件难看，而是页面定位不清。它现在同时混合了当前目标展示、任务偏好展示、今日任务入口、任务日历入口和页面说明标题，导致页面看起来像「个人状态页 + 快捷入口页 + 设置页」的混合体。

本报告建议将「我的」页重新定位为：

> 我的目标与偏好页。

也就是说，「我的」页主要负责长期信息管理，不负责当天任务执行。

---

## 2. 页面定位

「我的」页不应该和「今日任务页」「任务日历页」抢功能。

不同页面职责应该明确区分：

```text
今日任务页：今天做什么，如何执行。
任务日历页：目标计划如何排布，是否需要调整。
创建目标页：创建新的目标。
我的页：管理当前目标、默认偏好、AI 表达风格和个人设置。
```

因此，「我的」页的核心功能应该是：

1. 查看当前目标概览；
2. 管理目标；
3. 查看和编辑默认安排偏好；
4. 设置 AI 表达与仪式感偏好；
5. 查看最近推进数据；
6. 进入目标管理、复盘记录、偏好设置、关于反馈等设置项。

一句话总结：

> 我的页管理长期设置，不处理当天执行。

---

## 3. 当前页面存在的问题

### 3.1 页面主体重复标题

当前页面顶部导航栏已经显示「我的」，但页面主体中又出现：

- 「我的」胶囊标签
- 「个人状态」大标题
- 「这里只保留和任务安排相关的偏好。」说明文案

这些内容和顶部导航栏重复，也占用了首屏空间。建议删除。

### 3.2 当前目标卡片信息不足

当前目标卡片只显示了：

- 当前目标
- 目标名，例如 12334
- 截止日期
- 创建新目标按钮

问题是：

- 目标名缺少上下文；
- 没有显示目标进度；
- 没有显示计划状态；
- 有目标时仍然突出「创建新目标」，容易造成主次不清。

有目标时，用户更可能想查看计划或管理目标，而不是马上创建新目标。

### 3.3 任务偏好卡片过于粗糙

当前任务偏好以 2×2 小卡展示：

- 能量状态
- 偏好时段
- 专注时长
- MBTI

问题是：

- 信息像硬塞进去的状态块；
- 缺少编辑入口；
- MBTI 和任务安排硬参数放在同一级，容易让用户误解它会影响任务决策；
- 任务偏好和 AI 表达偏好没有区分。

建议将其拆成：

1. 默认安排偏好；
2. AI 表达与仪式感偏好。

### 3.4 页面底部跳转按钮重复

当前页面有两个按钮：

- 今日任务
- 任务日历

但底部 TabBar 已经有「今日」和「日历」，所以这两个按钮功能重复。建议删除。

## 4. 优化后的页面结构

建议将「我的」页整理成以下结构：

```text
顶部导航栏：我的

页面主体：
  用户问候区
  当前目标卡片
  默认安排偏好卡片
  AI 表达与仪式感偏好卡片
  最近推进卡片
  设置入口列表

底部 TabBar：
  今日 / 日历 / 创建 / 我的
```

其中，MVP 第一版可以优先实现前三到四块：

1. 当前目标卡片
2. 默认安排偏好卡片
3. AI 表达与仪式感偏好卡片
4. 设置入口列表

「最近推进」可以先做轻量展示，如果数据字段暂时不足，也可以先用 mock 或后续再接入。

## 5. 详细 UI 设计说明

### 5.1 顶部导航栏

保留现有顶部导航栏。

要求：

- 中间显示「我的」；
- 页面主体不再出现「我的」胶囊标签；
- 页面主体不再出现「个人状态」大标题；
- 页面主体不再出现页面说明副标题。

顶部导航栏已经承担页面身份说明，不需要重复。

### 5.2 用户问候区

参考新版图片，可以在页面顶部放一个轻量问候区。

示例文案：

```text
嗨，今天也要好好推进目标哦 🌱
小目标的积累，会带来大改变。
```

设计要求：

- 可以包含头像；
- 不要占用太多高度；
- 不要写成长文；
- 不要做成强功能入口；
- 右侧可以预留进入个人资料或设置的箭头。

如果项目暂时没有用户昵称和头像，可以使用默认头像和通用文案。

### 5.3 当前目标卡片

当前目标卡片是「我的」页的核心卡片。

#### 有目标状态

卡片标题：

```text
当前目标
```

展示内容：

```text
完成开题报告初稿
截止日期 2026-05-24｜还剩 2 天
整体进度 3 / 8 个任务
进度百分比 38%
计划状态：进行中
```

操作按钮：

```text
查看计划
管理目标
```

设计要求：

- 目标名称必须出现在有上下文的卡片中；
- 不要单独裸露显示目标名；
- 有目标时，「创建新目标」不应该作为最大按钮；
- 更重要的操作应该是「查看计划」和「管理目标」；
- 可使用进度条展示目标整体进度；
- 右上角可以显示目标状态标签，例如「进行中」「有风险」「已完成」。

字段建议：

```ts
currentGoal.title
currentGoal.deadline
currentGoal.daysLeft
currentGoal.status
currentGoal.progressPercent
currentGoal.completedTaskCount
currentGoal.totalTaskCount
```

按钮逻辑：

```text
查看计划 -> 跳转任务日历页，并定位到当前目标计划
管理目标 -> 进入目标管理页或目标详情页
```

#### 无目标状态

如果用户当前没有目标，不展示空白卡片，而是展示引导创建目标。

示例：

```text
还没有目标
先创建一个目标，我会帮你生成任务计划。
创建目标
```

设计要求：

- 无目标时可以突出「创建目标」按钮；
- 有目标时不突出「创建新目标」按钮；
- 避免让用户误以为必须创建多个目标才能使用产品。

按钮逻辑：

```text
创建目标 -> 跳转创建目标页
```

### 5.4 默认安排偏好卡片

这张卡用于展示系统安排任务时使用的默认参数。

卡片标题：

```text
默认安排偏好
```

右上角：

```text
编辑 >
```

建议展示：

```text
每天可用时间：30 分钟
偏好时段：都可以
计划强度：均衡
当前能量状态：普通
```

设计形式：

- 可以使用 4 个轻量图标摘要块；
- 或使用列表式摘要；
- 不要使用过深的背景块；
- 不要让它看起来像复杂设置页。

说明文案：

```text
系统将根据以上偏好为你安排每日任务。
```

字段建议：

```ts
userPreferences.defaultAvailableMinutes
userPreferences.preferredTime
userPreferences.planIntensity
userPreferences.energyState
```

编辑入口逻辑：

```text
编辑 -> 偏好设置页 / 底部弹窗 / 我的偏好管理页
```

### 5.5 AI 表达与仪式感偏好卡片

这一块用于承接后续的 AI 解读、每日关键词、塔罗灵感、MBTI 等个性化元素。

卡片标题：

```text
AI 表达与仪式感偏好
```

右上角：

```text
编辑 >
```

建议展示为列表：

```text
建议风格        温和鼓励
每日关键词      已开启
塔罗灵感        已开启
MBTI            未填写
```

底部说明：

```text
以上内容仅用于表达风格和仪式感，不作为科学预测依据。
```

设计边界：

- 不要把 MBTI、塔罗、关键词放入「默认安排偏好」的硬参数中；
- 不要暗示塔罗或 MBTI 会决定任务安排；
- 这些元素只影响文案表达、仪式感和提醒风格；
- 任务计划仍应主要依据目标、DDL、每日可用时间、任务完成情况和复盘状态。

字段建议：

```ts
aiPreferences.suggestionStyle
aiPreferences.dailyKeywordEnabled
aiPreferences.tarotHintEnabled
aiPreferences.mbti
```

建议风格选项：

```text
直接一点
温和鼓励
轻松陪伴
```

仪式感元素选项：

```text
每日关键词
塔罗灵感
都不需要
```

### 5.6 最近推进卡片

最近推进卡片用于展示用户近期努力成果，但不要做得太复杂。

卡片标题：

```text
最近推进
```

建议展示：

```text
本周完成任务：3 个
累计专注时间：120 分钟
连续推进天数：2 天
```

设计要求：

- 只展示 2-3 个核心指标；
- 不要做成复杂数据看板；
- 如果暂时没有真实数据，可以先隐藏或使用 mock，但需要注明；
- 不要让它比当前目标卡片更抢眼。

字段建议：

```ts
stats.weeklyCompletedTaskCount
stats.totalFocusMinutes
stats.continuousActiveDays
```

注意：这块不是 MVP 必需。如果当前数据还不稳定，可以先延后实现。

### 5.7 设置入口列表

设置入口列表建议放在页面底部。

建议入口：

```text
目标管理
复盘记录
偏好设置
关于与反馈
```

可选入口：

```text
数据清理
通知设置
隐私说明
```

MVP 阶段建议先保留 4 个入口即可，避免页面过重。

设计要求：

- 列表样式简洁；
- 每项左侧图标、右侧箭头；
- 不要每个入口都做成大卡；
- 不要放「今日任务」「任务日历」这种 TabBar 已有入口。

## 7. 交互设计

### 7.1 查看计划

点击「查看计划」：

```text
跳转任务日历页
定位到当前目标对应的 active Plan
```

### 7.2 管理目标

点击「管理目标」：

可以进入目标管理页，展示：

- 当前目标详情
- 修改目标名称
- 修改 DDL
- 暂停目标
- 取消目标
- 创建新目标
- 切换目标

MVP 中如果还没有目标管理页，可以先进入目标详情弹窗或保留占位说明。

### 7.3 编辑默认安排偏好

点击默认安排偏好卡右上角「编辑」：

进入偏好设置页或弹窗，允许修改：

- 默认每日可用时间
- 偏好工作时段
- 默认能量状态
- 计划强度

保存后更新「我的」页摘要。

### 7.4 编辑 AI 表达偏好

点击 AI 表达卡右上角「编辑」：

进入 AI 表达偏好设置页或弹窗，允许修改：

- 建议风格
- 是否开启每日关键词
- 是否开启塔罗灵感
- MBTI

保存后更新「我的」页摘要。

### 7.5 设置入口

点击设置列表项：

```text
目标管理 -> 目标管理页
复盘记录 -> 复盘历史页
偏好设置 -> 偏好设置页
关于与反馈 -> 关于反馈页
```

如果某些页面暂未实现，可以先用 toast 或占位页，但需要在开发回传中说明。

## 8. 视觉规范建议

### 8.1 整体风格

延续当前小程序风格：

- 暖白 / 米色背景；
- 白色圆角卡片；
- 紫色作为主色；
- 浅紫作为选中态和强调色；
- 橙色用于 DDL 风险或提醒；
- 绿色用于完成或积极反馈；
- 灰色用于辅助说明；
- 卡片之间保持统一间距。

### 8.2 当前目标卡片

当前目标卡片可以比其他卡片更突出。

建议：

```text
浅紫边框
白色或轻渐变背景
目标名称加粗
状态标签放右上角
进度条使用紫色
主按钮使用紫色
次按钮使用描边样式
```

### 8.3 默认偏好卡片

建议使用轻量图标摘要块。

例如：

```text
每天可用时间   30 分钟
偏好时段       都可以
计划强度       均衡
当前能量       普通
```

注意：

- 图标可以增加识别度；
- 文案要短；
- 不要过度装饰；
- 不要把 MBTI 放在这里。

### 8.4 AI 表达与仪式感卡片

建议使用列表样式。

例如：

```text
建议风格        温和鼓励 >
每日关键词      已开启 >
塔罗灵感        已开启 >
MBTI            未填写 >
```

底部说明使用浅灰色小字。

### 8.5 设置列表

建议放在单张白色圆角卡片中。

每一行：

```text
图标 + 标题 + 右箭头
```

行高保持舒适，不需要过多说明文字。

1. 不重构整个小程序。

3. 不重新设计今日任务页、任务日历页、创建目标页。

## 11. 验收标准

完成后请确保：

1. 页面顶部只保留顶部导航栏中的「我的」。
2. 页面主体不再出现「我的」胶囊标签。
3. 页面主体不再出现「个人状态」大标题。
4. 页面主体不再出现页面说明副标题。
5. 删除「今日任务」「任务日历」两个重复跳转按钮。
6. 页面第一张核心卡片是「当前目标」或「无目标引导」。
7. 当前目标卡片包含目标名称、截止日期、剩余天数、进度、状态。
8. 有目标时，主操作是「查看计划」「管理目标」，不是「创建新目标」。
9. 无目标时，突出「创建目标」按钮。
10. 默认安排偏好卡包含每日可用时间、偏好时段、计划强度、能量状态。
11. AI 表达与仪式感偏好单独成卡，不能混入默认安排偏好。
12. AI 表达卡明确说明只影响表达风格和仪式感，不作为预测依据。
13. 最近推进卡最多展示 2-3 个核心指标。
14. 设置入口列表简洁，不堆叠过多功能。
15. 页面整体比旧版更干净，更像「我的目标与偏好」页。
