请对微信小程序的「任务日历」页做 UI 和交互优化。本轮讨论建立在一个前提上：页面内部原来的冗余标题区已经删除，顶部导航栏已经负责显示当前页名称「任务日历」，底部 TabBar 已经负责页面切换。

一、页面定位

「任务日历」页不是另一个「今日任务」页。

今日任务页的核心是：告诉用户今天最应该做什么，并辅助用户执行。

任务日历页的核心是：展示一个目标从创建到截止日期之间的整体计划排布，包括任务分布、计划进度、时间压力、某一天的任务详情，以及重新调整计划的入口。

请把「任务日历」页设计成一个“目标计划板”，而不是普通日历，也不是任务列表堆叠页。

二、产品背景

本小程序是一个个性化 AI 任务日历。用户创建目标时，会输入：
- 目标名称
- 截止日期
- 每日可用时间
- 状态偏好

系统需要根据这些信息，把大目标拆成每天可执行的小任务，并生成每日任务日历。之后系统会根据用户的任务完成情况、当天状态和晚间复盘，动态调整后续计划。

因此，日历页要重点体现：
1. 当前目标是什么。
2. 离截止日期还有多久。
3. 当前目标整体完成了多少。
4. 未来几天怎么安排。
5. 某一天有哪些任务。
6. 如果计划太紧或任务延期，用户可以重新调整计划。

三、整体页面结构

请将「任务日历」页整理成以下结构：

1. 顶部导航栏
   - 保留现有顶部导航栏。
   - 中间显示「任务日历」。
   - 不要在页面主体里重复出现「任务日历」大标题、胶囊标签、页面说明副标题。

2. 当前目标计划卡
   - 页面主体第一张卡。
   - 展示当前目标和截止日期。
   - 让用户一眼知道当前查看的是哪个目标的计划。

3. 计划进度 / 时间压力卡
   - 展示当前目标的整体计划进度。
   - 注意：这里的进度不是单个任务进度，也不是所有目标总进度，而是“当前目标”的整体计划完成度。

4. 日期排布视图
   - 默认展示未来 7 天计划概览。
   - 如果目标截止日期超过 7 天，需要提供「周视图 / 月视图」切换。
   - 周视图看最近 7 天的详细任务安排。
   - 月视图看从当前日期到截止日期之间的整体排布、阶段节点、缓冲日和风险日。

5. 选中日期任务列表
   - 用户点击某一天后，在下方展示该日期的任务。
   - 这里展示的是某一天的任务详情，但不要把它做得比今日任务页更重。

6. AI 计划建议卡
   - 展示简短的计划建议。
   - 用于提示任务是否过密、是否有延期风险、是否建议重排。

7. 调整计划入口
   - 放在页面底部或作为醒目的轻量卡片。
   - 用于修改每日可用时间、截止日期、计划强度，或重新生成计划。

四、当前目标计划卡设计

当前目标计划卡需要包含：

- 小标签：当前目标
- 目标名称，例如：完成开题报告初稿
- 截止日期，例如：截止日期 2026-05-24
- 距离截止，例如：还剩 2 天
- 总任务数，例如：共 8 个任务
- 预计剩余时间，例如：预计还需 120 分钟
- 计划状态，例如：
  - 进度正常
  - 计划偏紧
  - 有延期风险
  - 已完成
- 目标切换入口，例如：切换目标 >

如果当前只有一个目标，可以保留「切换目标」按钮但弱化显示；如果项目暂时不支持多目标切换，可以先不实现切换逻辑，但 UI 位置可以预留，或者隐藏该入口。

不要再出现页面顶部裸露的目标名，例如单独显示一个「12334」。目标名必须出现在有上下文的卡片里。

五、计划进度 / 时间压力卡设计

计划进度卡展示的是“当前目标”的整体计划进度。

不要显示无上下文的裸数字。每个数字都要有标签说明。

建议展示：

1. 整体进度
   - 使用横向进度条。
   - 例如：整体进度 57%。

2. 已完成任务
   - 例如：3 / 8。

3. 已完成天数
   - 例如：4 / 7。

4. 预计剩余时间
   - 例如：120 分钟。

5. 距离截止日期
   - 例如：还剩 2 天。

MVP 阶段，整体进度可以先用简单算法：

整体进度 = 已完成任务数 / 总任务数

如果现有数据里有任务预计时长，更推荐：

整体进度 = 已完成任务预计时长 / 全部任务预计时长

请优先复用现有数据。如果暂时没有完整字段，可以先用 mock 数据或计算兜底，但需要在回传说明中明确哪些数据是真实的，哪些是 mock 的。

注意：
- 单个任务的状态不要放在这个大进度条里。
- 单个任务只在任务卡里显示：未开始 / 进行中 / 已完成 / 延期 / 已重排。
- 计划进度卡只服务于当前目标整体计划。

六、日期排布视图设计

日期排布视图是日历页的核心。

请不要一开始就把所有任务全部铺成一个长列表。更好的方式是：

默认展示「未来 7 天计划概览」。

每个日期小卡展示：
- 星期
- 日期
- 当天任务数量
- 当天预计用时
- 当天状态

示例：

05/23 今天
1 个任务
30 分钟

05/24 明天
2 个任务
60 分钟

05/25
休息日

05/26
1 个任务
45 分钟

日期状态建议：
- 紫色：有任务
- 灰色：无任务
- 绿色：已完成
- 橙色：缓冲日或任务偏重
- 红色：延期风险或任务超载

日期小卡需要支持点击。点击后，下方的「选中日期任务列表」刷新为该日期的任务。

七、周视图 / 月视图逻辑

如果当前目标的截止日期在 7 天以内：
- 默认展示未来 7 天即可。
- 可以不突出月视图。

如果当前目标的截止日期超过 7 天：
- 默认仍然展示未来 7 天详细计划。
- 增加「周视图 / 月视图」切换。
- 周视图展示最近 7 天的具体任务。
- 月视图展示从当前日期到截止日期的整体排布。

月视图不要把 30 天任务全部写成密密麻麻的文字。建议展示：
- 有任务的日期用小圆点标记。
- 缓冲日用浅橙色标记。
- 阶段节点用小旗帜或标签标记。
- 截止日期用醒目标识。
- 有延期风险的日期用红点或警示图标标记。

点击月视图中的某一天，下方仍然展示该日期任务详情。

八、长期目标的计划生成逻辑

如果目标截止日期在一个月后，不要一次性把未来 30 天全部细化成固定小任务。

请采用「近详远略」的计划方式：

1. 未来 7 天
   - 生成具体到每天的小任务。
   - 每天有明确任务、预计时长、最低完成线。

2. 7 天之后到截止日期
   - 先生成阶段计划或粗粒度节点。
   - 例如：
     - 第 2 周：完成初稿主体
     - 第 3 周：补充案例和实验设计
     - 第 4 周：修改、格式整理、预留缓冲

3. 每天复盘之后
   - 根据用户完成情况和当天状态，滚动细化后续计划。
   - 未完成任务可以被重排到后续日期。

UI 表达上：
- 最近 7 天展示具体任务。
- 更远的日期展示阶段节点、缓冲日、DDL、风险状态。
- 如果某一天还没有被细化，可以显示：
  「阶段计划：完成背景部分」
  或
  「AI 将在临近时细化任务」

不要让用户误以为未来一个月每天的任务都是不可变的。这个产品的计划应该是动态更新的。

九、选中日期任务列表设计

用户点击某一天后，页面下方展示该日期的任务列表。

标题示例：

05/24 星期六
预计 60 分钟｜2 个任务

任务卡字段：

- 任务标题，例如：整理已有材料
- 所属目标，例如：完成开题报告初稿
- 标签：重点任务 / 普通任务 / 低能量备用
- 最低完成线，例如：把已有资料分成 3 类
- 预计时长，例如：30 分钟
- 优先级，例如：中优先级
- 状态：未开始 / 进行中 / 已完成 / 延期
- 操作按钮：
  - 查看详情
  - 调整日期
  - 移到今天
  - 标记完成

注意：
- 日历页不是主要执行页，所以「开始专注」按钮可以有，但不应该比今日任务页更突出。
- 如果要突出执行，请优先引导用户去今日页。
- 日历页更重要的是查看排布和调整计划。

十、AI 计划建议卡

AI 计划建议卡要短，不要做成长文解读。

建议结构：

标题：AI 计划建议

正文示例：
- 当前计划略紧，建议保留 1 天缓冲。
- 05/24 有 2 个任务，建议先完成重点任务的最低完成线。
- 如果今天未完成，可以将低优先级任务顺延到明天。

右侧可以有「查看详情 >」，但默认不要展开太多。

注意：
MBTI、塔罗、每日关键词等只作为表达风格和仪式感包装，不作为科学预测依据。日历页可以出现轻量关键词，但不要让塔罗或 MBTI 决定任务排布。

可选轻量表达：
今日节奏关键词：节制
建议：不要把所有任务压到最后一天，保留缓冲。

不要出现复杂塔罗牌阵、运势日历、人格分析长文。

十一、调整计划入口

日历页需要有一个明显但不喧宾夺主的「调整计划」入口。

入口文案建议：
调整计划
修改每日可用时间、截止日期或重新生成计划

点击后可以进入一个二级页或弹窗。

调整计划页 / 弹窗建议包含：

1. 截止日期
   - 用户可以修改目标截止日期。

2. 每日可用时间
   - 例如：15 / 30 / 45 / 60 / 90 分钟
   - 支持自定义分钟数。

3. 计划强度
   - 轻松
   - 均衡
   - 冲刺

4. 是否保留缓冲日
   - 开 / 关
   - 默认建议开启。

5. 优先安排时段
   - 上午
   - 下午
   - 晚上
   - 都可以

6. 重新生成计划按钮
   - 点击后根据新参数重新排布后续任务。

注意：
本轮可以先完成 UI 和本地状态更新，不要求完整接入真实 AI 重排。如果只做 mock，请明确说明。

十二、特殊状态处理

请处理以下状态：

1. 无目标
   - 显示空状态。
   - 文案：还没有目标，先创建一个目标，我会帮你生成任务日历。
   - 按钮：创建目标。

2. 有目标但无计划
   - 显示当前目标。
   - 提示：还没有生成计划。
   - 按钮：生成计划。

3. 截止日期在 7 天以内
   - 展示未来 7 天计划。
   - 强调剩余天数和时间压力。

4. 截止日期超过 7 天
   - 默认展示未来 7 天详细计划。
   - 提供月视图查看长期排布。

5. 某一天无任务
   - 显示：这天暂无任务 / 缓冲日。
   - 如果是缓冲日，要明确标注。

6. 某一天任务过多
   - 显示任务偏重或计划偏紧提示。
   - AI 计划建议中提示可以调整。

7. 有延期任务
   - 在日期视图中标记风险。
   - 在任务列表中显示延期状态。
   - 提供重排入口。

8. 目标已完成
   - 显示完成状态。
   - 日期视图可以保留历史任务。
   - 调整计划入口弱化。
