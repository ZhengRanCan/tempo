请对当前微信小程序做第一轮 UI 清理，这一轮只处理“四个底部 Tab 页面中的页内标题区冗余问题”，不要重构业务逻辑，也不要大改具体卡片内容。

背景：
当前小程序已经有顶部导航栏用于显示当前页面名称，也有底部 TabBar 用于切换页面。因此页面内部不应该再重复出现胶囊标签、大标题、副标题、裸露目标名等“伪标题区”。这些内容导致首屏顶部空间浪费，页面显得冗杂。

需要保留：
1. 顶部导航栏，继续显示当前页名称，例如「今日任务」「任务日历」「创建目标」「我的」。
2. 底部 TabBar，继续用于四个页面切换。
3. 页面主体中的真实业务组件，例如当前目标卡片、任务偏好卡片、创建目标表单、日历任务列表、今日任务空状态等。

需要删除或调整：
1. 删除页面主体顶部的小胶囊标签，例如「我的」「Tempo」「任务日历」「今日任务」。
2. 删除页面主体顶部的大标题，例如「个人状态」「创建目标」「今日任务」等。
3. 删除大标题下方的说明性副标题，例如：
   - 「这里只保留和任务安排相关的偏好。」
   - 「先写下一个想推进的目标，再选一个从今天开始还来得及完成的截止日期。」
   - 「先看最近 7 天，每天只保留能执行的小步骤。」
   - 「只看今天最值得推进的一小步。」
4. 删除页面顶部裸露展示的「12334」这类目标名/数字。注意：如果它是目标名称，可以保留在有上下文的组件中，例如「当前目标」卡片、任务卡片、日历任务项里，但不要作为页面顶部独立标题展示。
5. 删除标题区残留造成的大块顶部留白。页面主体内容应该在顶部导航栏下方自然开始，建议保留 24rpx-32rpx 的基础间距即可。
