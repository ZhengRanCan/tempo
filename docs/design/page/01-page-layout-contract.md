# 页面布局契约

## 目的

本文件只定义四个 Tab 页的页面级布局职责，不定义颜色、字号、圆角、按钮、标签、空状态等视觉细节。

视觉细节统一参考：

- `docs/harness/DESIGN.md`
- `docs/design/visual-system.md`
- `docs/design/components.md`
- `docs/design/interaction-states.md`
- `docs/design/content-rules.md`

## 顶部导航栏契约

- MVP 必须：顶部导航栏负责显示当前页面名称。
- MVP 必须：页面主体不得重复当前页面大标题。
- MVP 必须：页面主体不得出现页面级胶囊标签。
- MVP 必须：导航栏下方直接进入当前页面的核心内容。
- 待确认：是否保留微信胶囊按钮旁的自定义返回按钮，由具体页面栈决定。

## 底部 TabBar 契约

- MVP 必须：底部 TabBar 只负责四个主页面切换。
- MVP 必须：四个主入口为 今日 / 日历 / 创建 / 我的。
- MVP 必须：今日任务页是第一入口。
- MVP 必须：页面主体不得把 TabBar 已有入口重复做成大按钮。
- 后续扩展：复盘详情、目标管理、计划调整、今日全部任务等非 Tab 页面，通过页面内入口进入。

## 四个 Tab 页首屏契约

| 页面 | 首屏第一核心内容 | 不应出现 |
|---|---|---|
| 今日任务 | 今日重点任务卡；无任务时显示今日空状态 | 完整目标计划、未来 7 天列表 |
| 任务日历 | 当前目标计划卡 | 今日执行主按钮堆叠 |
| 创建目标 | 目标创建步骤卡 | 任务列表、计划详情 |
| 我的 | 当前目标卡或无目标引导卡 | 今日任务 / 任务日历重复入口 |

## 页面主体顶部间距

- MVP 必须：导航栏下方只保留基础安全间距。
- 建议：24rpx-32rpx。
- 禁止：大块顶部空白。
- 禁止：页面主体先出现无业务价值的大标题区。

## 页面职责边界

- 今日任务页只服务当天执行。
- 任务日历页只服务目标计划排布查看和调整。
- 创建目标页只服务 Goal 创建和 Plan 生成触发。
- 我的页只服务目标、偏好和设置管理。

## 数据概念引用

页面文档可以说明本页面读取哪些数据，但不在本目录解释完整数据模型。

Goal、Plan、Stage、Task 的完整边界默认引用架构侧文档：

```text
docs/architecture/goal-plan-task-state-model.md
```

当前本轮不创建或修改该架构文档。

## 禁止项

- 页面主体重复标题。
- 页面主体顶部胶囊标签。
- 大块顶部空白。
- 裸露目标名。
- 裸露统计数字。
- 把 TabBar 已有入口重复做成按钮。
- 把今日页做成完整计划页。
- 把日历页做成今日执行页。
- 把我的页做成任务执行页。
- 将塔罗、MBTI、每日关键词作为任务决策依据。
- 把 Goal、Plan、Stage、Task 混写成一个概念。
