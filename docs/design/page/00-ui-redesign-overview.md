# UI 改造总览

## 为什么要大改 UI

v0.1 已经完成目标创建、计划生成、今日任务、复盘和重排闭环，但页面 UI 仍然带有明显的早期堆叠痕迹：

- 顶部导航栏已经显示页面名称，页面主体仍重复出现标题。
- 任务、计划、目标信息混在同一层级展示。完整数据边界由架构侧文档负责，本目录只处理页面呈现。
- 用户打开页面后，不容易马上判断当前页面要做什么。
- 部分卡片承担了过多解释、入口和状态展示职责。
- AI、塔罗、MBTI、每日关键词等仪式感元素容易被误读为任务决策依据。

本轮设计整理的目标是为后续 UI 修改建立统一施工规范，不直接修改业务代码。

## 当前 UI 的共同问题

- 页面主体重复标题：页面内重复出现胶囊标签、大标题、长副标题。
- 顶部空间浪费：导航栏下方留白过大，首屏核心内容下移。
- 信息层级不清：目标名、统计数字、任务卡片缺少上下文。
- 页面职责混杂：今日页、日历页、我的页之间存在入口和职责重复。
- 组件过多或过重：卡片解释文字偏长，用户需要阅读过多说明。
- 概念混用：Goal、Plan、Stage、Task 在 UI 和字段描述中容易混写。

## 全局改造原则

- 顶部导航栏负责页面名称。
- 底部 TabBar 负责页面切换。
- 页面主体不要重复大标题、胶囊标签和长副标题。
- 首页首屏尽量只展示当前页面最核心任务。
- AI、塔罗、MBTI、每日关键词只作为表达风格和仪式感，不作为科学预测依据。
- 页面呈现必须区分 Goal、Plan、Stage、Task；完整模型定义引用 `docs/architecture/goal-plan-task-state-model.md`。
- 目标名必须放在有上下文的组件中，不得裸露在页面顶部。
- 统计数字必须有标签，不得裸露展示。

## 四个页面的目标改造方向

| 页面 | 改造方向 |
|---|---|
| 今日任务页 | 改成执行台，首屏突出一个今日重点任务 |
| 任务日历页 | 改成目标计划板，展示当前目标整体计划排布 |
| 创建目标页 | 改成步骤式 Goal 创建流程 |
| 我的页 | 改成我的目标与偏好管理页 |

## 本轮不做的事情

- 不修改小程序页面代码。
- 不修改组件代码。
- 不修改业务逻辑。
- 不修改数据结构代码。
- 不修改路由。
- 不修改 TabBar 配置。
- 不删除参考图片。
- 不把塔罗、MBTI、每日关键词写成任务决策依据。
- 不把 Goal、Plan、Stage、Task 混写成一个概念。
- 不将文档整理成泛泛的产品愿景。
