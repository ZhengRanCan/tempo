# F25 今日页首屏视觉参考

## 1. Purpose

本文档用于指导 F25 对今日任务页进行首屏视觉优化。

F25 的目标不是重新设计今日页完整业务结构，而是在已有今日页结构基础上，提升首屏信息密度、按钮比例、图标引导、卡片节奏和参考图对齐程度。

本文档只作为 F25 的视觉参考，不替代 `feature.md` 和 `verification.md`。

## 2. Reference Goal

参考图不是颜色参考，而是结构参考。

F25 应重点参考：

* 首屏模块数量；
* 信息层级；
* 卡片密度；
* 图标引导；
* 主次按钮比例；
* 首屏内容完整度；
* 任务执行感。

不要求逐像素一致。

但如果只是修改颜色、圆角、阴影，而没有改善按钮比例、图标锚点、首屏密度和模块完整度，不能认为已接近参考图。

## 3. Current Gap

当前今日页已经具备基础模块，但与参考图仍有明显差距：

* 主按钮比例和视觉权重不足；
* 次级操作与主操作层级不够清楚；
* 关键模块缺少统一图标锚点；
* AI 今日建议和晚间复盘入口容易落到首屏之外；
* 卡片高度、间距和字号层级导致首屏信息密度不足；
* 底部 TabBar 缺少图标，但该项不在 F25 中实现。

## 4. Button Rules

今日页按钮必须重新按小程序端校准，不直接套用旧统一按钮。

### 主按钮

主按钮用于当前页面最重要动作，例如：

* 开始专注；
* 生成计划；
* 去复盘。

今日页主按钮重点用于：

* `开始专注`

要求：

* 宽度明显大于普通按钮；
* 高度适合移动端点击；
* 圆角、字号、字重重新校准；
* 文字居中；
* 文字必须完整显示，不压缩、不换行、不拥挤；
* 视觉上强于次级操作；
* 可使用紫色实色或紫色渐变，但不得影响文字可读性。

### 次级按钮

次级按钮用于辅助动作，例如：

* 标记完成；
* 查看全部；
* 管理；
* 去复盘中的弱入口状态。

要求：

* 使用文字按钮、轻量描边按钮或低权重按钮；
* 不抢主按钮视觉权重；
* 不与主按钮做成两个同等级强按钮；
* 与主按钮放在同一区域时，必须能明显看出主次关系。

## 5. Icon Guidance

今日页需要为关键模块补充统一风格的视觉锚点图标。

图标用于增强扫描效率，不用于装饰堆叠。

建议图标语义：

| 模块 / 信息 | 建议图标                      |
| ------- | ------------------------- |
| 今日重点任务  | star                      |
| 预计时间    | clock                     |
| 优先级     | flag                      |
| 今日任务数量  | list                      |
| 今日状态    | activity / pulse / status |
| AI 今日建议 | sparkle                   |
| 晚间复盘    | moon                      |

## 6. Icon Style

图标必须风格统一。

要求：

* 优先使用同一来源、同一风格的线性图标；
* 图标粗细保持一致；
* 图标尺寸保持一致；
* 不混用 emoji、彩色插画、粗细不同的图标；
* 不让 agent 自行生成最终图标资源；
* 没有正式图标资源时，可以预留图标位或使用临时占位，但必须记录。

## 7. Icon Color

图标颜色应跟随页面语义，不要每个图标单独随意配色。

建议：

* 今日重点任务、今日关键词：使用暖橙色或品牌辅助色；
* 开始专注、选中状态、AI 建议：使用主紫色或品牌强调色；
* 预计时间、优先级等元信息：使用中性灰；
* 晚间复盘：使用紫色或低饱和深色；
* 错误、危险或破坏性操作才允许使用红色。

图标不应使用高饱和多色风格。

普通状态图标应能和文字、标签、按钮形成统一视觉，而不是成为单独抢眼的装饰。

## 8. Icon Asset Location

图标资源必须放在统一目录中，不要散落在页面目录或组件目录。

推荐目录：

```text
static/icons/
  today/
  tab/
  common/
```

今日页图标建议路径：

```text
static/icons/page/today/star.png
static/icons/page/today/clock.png
static/icons/page/today/flag.png
static/icons/page/today/list.png
static/icons/page/today/status-wave.png
static/icons/page/today/sparkle.png
static/icons/page/today/moon.png
```

## 9. First Screen Completeness

今日页首屏应尽量同时露出：

* 今日重点任务卡；
* 今日状态卡；
* AI 今日建议卡；
* 晚间复盘入口的至少一部分；
* 底部 TabBar。

如果 AI 今日建议和晚间复盘入口完全落到首屏之外，说明首屏信息密度仍未接近参考图。

允许通过以下方式提升首屏完整度：

* 减少卡片外边距；
* 减少无业务价值留白；
* 压缩重复文案；
* 调整卡片内边距；
* 调整字号层级；
* 合并过长说明；
* 将弱信息降级为短句或标签。

不得通过以下方式提升首屏完整度：

* 删除必要模块；
* 缩小到文字难以阅读；
* 压缩按钮导致文字换行或拥挤；
* 改变今日页核心业务结构；
* 修改数据模型或 services。

## 11. Acceptance Reference

F25 的完成判断应重点看：

* 主按钮和次级按钮层级是否明显；
* 按钮文字是否完整显示；
* 今日页关键模块是否具备统一图标锚点或预留图标位；
* 图标风格和颜色是否统一；
* 首屏信息密度是否接近参考图；
* AI 今日建议和晚间复盘入口是否不再完全被挤出首屏；
* 是否记录了图标资源来源或占位策略；
* 是否没有修改 models、services、storage key、planner、replanner、today-suggestion 业务逻辑。

如果只是改颜色、阴影、圆角，但首屏密度、按钮比例、图标锚点没有明显改善，不能通过。
